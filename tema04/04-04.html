4. TensorFlow y Keras: El poder de las redes neuronales profundas

TensorFlow ha revolucionado el desarrollo de inteligencia artificial en todo el mundo. Fue desarrollado originalmente por Google Brain y se ha convertido en el estándar de facto para proyectos de aprendizaje profundo. En 2019, la versión 2.0 integró Keras de manera oficial, lo que simplificó enormemente la programación de redes neuronales profundas gracias a sus módulos de alto nivel que abstraen la complejidad.

Ecosistema de TensorFlow

El ecosistema de TensorFlow en 2025 incluye herramientas complementarias muy útiles:

  • TensorBoard: para visualizar el proceso de entrenamiento y los resultados de forma interactiva
  • TensorFlow Lite: para ejecutar modelos en dispositivos móviles y embebidos con recursos limitados
  • TensorFlow.js: para correr modelos directamente en navegadores web
  • TensorFlow Extended (TFX): para desplegar modelos en producción a gran escala

Keras, ahora parte integral de TensorFlow, está diseñada pensando en los humanos que la programan. Su filosofía es hacer el código más intuitivo y reducir la cantidad de acciones necesarias para lograr resultados. Además, ofrece mensajes de error claros y comprensibles que te ayudan a identificar y corregir problemas rápidamente.

Flujo de trabajo típico con TensorFlow y Keras

1. Carga del dataset

Importas tus datos al programa, ya sea desde archivos locales o desde repositorios en línea.

2. Construcción del modelo

Utilizas la clase Sequential que permite apilar capas de neuronas una tras otra de forma secuencial. Añades capas usando la clase Dense (capas densamente conectadas) y especificas funciones de activación como relu (para capas intermedias) o softmax (para la capa final en problemas de clasificación múltiple).

3. Configuración de parámetros

Mediante el método compile, indicas qué optimizador usar (como Adam o SGD), qué función de pérdida o coste emplear (que mide qué tan equivocado está el modelo), y qué métrica monitorear (como accuracy para medir la precisión).

4. Preparación de datos

Ajustas las dimensiones de los arrays para que coincidan con lo que el modelo espera, normalizas los valores de píxeles (típicamente dividiéndolos entre 255 para que estén entre 0 y 1), y conviertes las etiquetas a formato categórico usando to_categorical.

5. Entrenamiento

Usas el método fit especificando el número de iteraciones completas sobre los datos (epochs) y el tamaño del lote (batch size), que determina cuántos ejemplos se procesan antes de actualizar los pesos del modelo.

6. Evaluación

Empleas el método evaluate con tus datos de test para obtener métricas de rendimiento sobre datos que el modelo nunca vio durante el entrenamiento.
