1.3.- Dialogflow: Tu asistente para crear conversaciones inteligentes

Dialogflow es una plataforma creada por Google Cloud que te permite construir aplicaciones capaces de entender y mantener conversaciones en lenguaje natural con las personas. Piensa en ella como el cerebro que hace posible que tu aplicación pueda hablar y entender a los usuarios de manera natural.

Esta herramienta está especialmente diseñada para crear interfaces conversacionales (como chatbots o asistentes virtuales) y lo mejor es que puedes integrarla en prácticamente cualquier cosa: aplicaciones móviles, páginas web, sistemas automatizados, robots e incluso en centros de atención al cliente donde trabajan personas reales. De hecho, existe una extensión llamada Agent Assist que permite que tu chatbot colabore con agentes humanos para dar mejor servicio.

Una de las funcionalidades más poderosas de Dialogflow es su capacidad de conectarse con servicios externos y realizar acciones reales. Por ejemplo, imagina que tienes un restaurante: cuando un cliente quiere reservar una mesa, Dialogflow puede consultar automáticamente tu base de datos de reservas y confirmarle si hay disponibilidad para el día y la hora que desea.

¿Cómo funciona Dialogflow paso a paso?

El flujo de trabajo es el siguiente:

  1. El usuario final escribe o habla su mensaje (por ejemplo, "quiero reservar mesa para dos personas")
  2. Dialogflow analiza el mensaje y busca coincidir con una intención (intent matching)
  3. Si es necesario obtener información adicional o realizar una acción, Dialogflow envía una solicitud (webhook request) a tu servicio
  4. Tu servicio puede entonces realizar acciones como consultar una base de datos o llamar a APIs externas
  5. El servicio devuelve la respuesta (webhook response) con la información solicitada
  6. Dialogflow prepara la respuesta final para el usuario
  7. El usuario recibe la respuesta en formato texto o voz

Formas de comunicación con Dialogflow

Dialogflow es muy flexible en cuanto a cómo puede recibir y enviar información. Puede aceptar entradas de texto (cuando escribes) o audio (cuando hablas), por ejemplo a través de Google Assistant o desde una llamada telefónica. Del mismo modo, puede responder mediante texto escrito o utilizando voz sintética (como cuando Siri o Alexa te hablan).

Esta plataforma forma parte de un conjunto más amplio de servicios que Google Cloud ofrece bajo el nombre "Conversational AI" (Inteligencia Artificial Conversacional). Estos servicios trabajan juntos para ayudarte a crear aplicaciones conversacionales completas y profesionales para tu negocio.

Por ejemplo, cuando le dices a tu teléfono "Hey Google, háblame sobre Kotlin", sucede lo siguiente:

  • Tu voz se convierte en texto (Speech to Text)
  • Se procesa el lenguaje natural para entender qué quieres (NLP)
  • Se identifica la intención correspondiente
  • Se busca la mejor respuesta
  • La respuesta se convierte en voz (Text to Speech) para que la escuches

Los dos pilares de un asistente conversacional

Para crear un asistente conversacional efectivo necesitas fundamentalmente dos cosas:

1. Identificar las intenciones del usuario: Cuando alguien hace una pregunta o solicita algo, tu asistente debe entender qué es lo que realmente quiere. Por ejemplo, si dice "hace frío aquí", puede estar pidiendo que subas la temperatura, no solo haciendo una observación.

2. Proporcionar respuestas adecuadas: Una vez que has entendido qué quiere el usuario, necesitas tener preparadas las respuestas apropiadas y seleccionar la más adecuada para cada situación.

Cómo empezar a crear tu agente en Dialogflow

El primer paso es crear tu agente (así se llama al asistente conversacional en Dialogflow). El proceso es sencillo:

1. Aceptas los términos de servicio. Es importante saber que al aceptar, también estás de acuerdo en seguir los términos de Firebase si decides usar esos servicios. Por defecto, tus datos de Firebase Analytics pueden ayudar a mejorar otros productos de Google, pero puedes controlar esto en la configuración cuando quieras.

2. Haces clic en "Create new agent" (Crear nuevo agente)

3. Configuras los detalles básicos:

  • Nombre del agente: El nombre que identificará a tu asistente
  • Idioma predeterminado: El idioma principal en el que trabajará (puedes añadir más idiomas después). En 2025, Dialogflow soporta más de 30 idiomas
  • Zona horaria: Para gestionar correctamente fechas y horas en las conversaciones
  • Proyecto de Google Cloud: Puedes vincular o crear un proyecto para acceder a funcionalidades avanzadas
  • Tipo de agente: Tienes la opción de crear un "Mega Agent" que puede combinar varios agentes más pequeños

Trabajando con Intenciones (Intents)

Las intenciones son el corazón de tu agente conversacional. Representan lo que el usuario quiere hacer o saber. Dialogflow viene con algunas intenciones predeterminadas que cubren situaciones comunes:

Default Welcome Intent: Se activa cuando el usuario saluda o inicia la conversación. Incluye frases como "hey", "hola", "qué tal", "buenas".

Default Fallback Intent: Se activa cuando el agente no entiende lo que el usuario quiere decir. Es importante tener respuestas amables y útiles como "¿No he entendido a qué te refieres?", "¿Podrías repetirlo por favor?", "¿Disculpa?", "Entiendo, ¿podrías ser más específico?".

Creando tus propias intenciones personalizadas

Una vez que tienes las intenciones básicas, puedes crear las tuyas propias siguiendo este proceso:

1. Añade frases de entrenamiento (Training Phrases o Utterances): Son ejemplos de cómo los usuarios podrían expresar esa intención. Por ejemplo, para una intención de "ReservaMesa", podrías incluir:

  • "llamaba para reservar una mesa para dos personas"
  • "tienen mesa para dos hoy a mediodía?"
  • "tienen disponibilidad para comer hoy dos personas?"
  • "hay disponible una mesa para hoy?"
  • "Querría una mesa para dos"
  • "quiero reservar una mesa"

Se recomienda incluir entre 10 y 15 frases diferentes por cada intención. Esto ayuda al sistema a aprender mejor y reconocer diferentes formas de expresar lo mismo.

2. Define los parámetros necesarios: Son los datos que necesitas del usuario para completar la acción. Por ejemplo, para reservar mesa necesitas:

  • Número de personas (number)
  • Fecha y hora (date-time)
  • Restaurante (si tienes varios)
  • Ubicación (location, opcional)

Puedes marcar parámetros como REQUERIDOS, lo que significa que si el usuario no los menciona, el agente le preguntará automáticamente. Por ejemplo: "¿Para cuántas personas?", "¿Para qué fecha y hora?", "¿En qué restaurante?"

3. Crea las respuestas: Define qué debe decir el agente cuando identifica esta intención y tiene toda la información necesaria. Puedes usar variables para personalizar las respuestas con los datos proporcionados por el usuario:

  • "Su reserva en $restaurante el día $date-time ha sido confirmada"
  • "Reserva realizada para el día $date-time"

Puedes incluir varias variantes de respuesta para que las conversaciones sean más naturales y no siempre diga exactamente lo mismo.

En 2025, Dialogflow continúa siendo una de las plataformas más utilizadas para crear asistentes conversacionales, y se ha integrado aún más profundamente con el ecosistema de Google Cloud, ofreciendo mejores capacidades de inteligencia artificial y procesamiento de lenguaje natural.
