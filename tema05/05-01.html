Programación de Redes Neuronales Profundas con Keras

1. Arquitectura de Modelos Sequential

1.1 El Perceptrón: La Unidad Fundamental

La unidad básica de cualquier red neuronal es el perceptrón. Matemáticamente, realiza una transformación lineal seguida opcionalmente de una función de activación no lineal. La ecuación fundamental es:

y = f(∑i=1n wi xi + b) = f(WT X + b)

Donde:

  • xi: Variables de entrada (features)
  • wi: Pesos (weights) que determinan la importancia de cada entrada
  • b: Sesgo (bias), un término independiente que permite desplazar la función
  • f: Función de activación que introduce no linealidad

El proceso de aprendizaje consiste en ajustar los valores de wi y b para minimizar el error entre las predicciones y los valores reales.

1.2 Modelo Sequential en Keras

El tipo de modelo más simple es el modelo Sequential, que es una pila lineal de capas. Para arquitecturas más complejas, puedes usar la API funcional de Keras, que permite construir grafos arbitrarios de capas, o usar subclassing para escribir modelos desde cero.

Las estructuras de datos principales de Keras son las capas y los modelos. Una capa es una transformación simple de entrada/salida, y un modelo es un grafo acíclico dirigido (DAG) de capas. La clase tf.keras.layers.Layer es la abstracción fundamental en Keras. Una capa encapsula un estado (pesos) y algún cálculo (definido en el método call).

import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

# Modelo básico con Sequential
model = keras.Sequential([
    layers.Dense(128, activation='relu', input_shape=(18,)),
    layers.Dense(64, activation='relu'),
    layers.Dense(1, activation='sigmoid')
])

# Alternativa: añadir capas una por una
model = keras.Sequential()
model.add(layers.Dense(128, activation='relu', input_shape=(18,)))
model.add(layers.Dense(64, activation='relu'))
model.add(layers.Dense(1, activation='sigmoid'))

1.3 Visualización de la Arquitectura

Keras proporciona métodos útiles para inspeccionar la estructura del modelo:

# Resumen del modelo
model.summary()

# Visualización gráfica (requiere graphviz y pydot)
keras.utils.plot_model(model, show_shapes=True, show_layer_names=True)

Resumen de Modelos

Modelo Descripción Cuándo usarlo Ejemplo
Sequential Pila lineal de capas donde cada capa tiene exactamente una entrada y una salida Arquitecturas simples donde los datos fluyen secuencialmente de una capa a la siguiente keras.Sequential([layers.Dense(64), layers.Dense(1)])
Functional API Permite crear grafos arbitrarios de capas con múltiples entradas/salidas y conexiones complejas Modelos con ramificaciones, múltiples entradas/salidas, capas compartidas o conexiones residuales output = layers.Dense(1)(x) seguido de keras.Model(inputs, outputs)
Subclassing Definir modelos como clases Python heredando de keras.Model, con control total sobre el forward pass Arquitecturas muy personalizadas o investigación donde se necesita máxima flexibilidad Clase que hereda de keras.Model con método call()
